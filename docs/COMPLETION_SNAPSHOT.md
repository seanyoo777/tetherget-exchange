# 전체 서비스 완성도 스냅샷

**“실제로 열 수 있는” 수준**(실계정·DB 원장·보안·매칭 신뢰도·관측·복구)을 100%로 두고, 아래는 **주관적이지만 코드베이스 기준**으로 맞춘 값입니다.  
**매시간** Cursor/에이전트가 자동으로 숫자를 올려 줄 수는 없으므로, **정각마다** 이 파일 또는 `completion-status.json`을 갱신하거나 `npm run report:completion` 을 실행하는 방식을 권장합니다.

### 가중 합성 (권장 가중치)

- 프론트 **30%**, 백엔드 **35%**, DB **25%**, 배포·운영 **10%**

```
0.30×58 + 0.35×40 + 0.25×12 + 0.10×55 = 17.4 + 14 + 3 + 5.5 = 39.9 ≈ 40%
```

## 갱신 방법

1. 진행 상황 반영 후 `completion-status.json` 수정
2. `npm run report:completion` 실행 → 콘솔에 요약 출력
3. (선택) Windows 작업 스케줄러 / cron 으로 시간당 `report:completion` 로그 적재

상세 성숙도는 `docs/SERVICE_MATURITY.md` 참고.
