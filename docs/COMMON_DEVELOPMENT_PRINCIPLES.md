# 공통 개발 원칙

모든 프로젝트는 추후 **하나의 통합 생태계**로 연결될 수 있도록 아래 원칙을 따른다.

---

## 1. 통합 생태계 지향 구조

모든 프로젝트는 추후 하나로 묶일 수 있도록 **경계·모듈·식별자**를 설계한다.

## 2. 회원 시스템 통합 가능 구조

- 하나의 계정으로 여러 플랫폼 사용을 고려한다.  
- **공통 `user_id`** 사용.  
- **공통 레퍼럴 구조**를 수용할 수 있게 설계한다.

## 3. 관리자 시스템 공통 구조

권한 분리를 전제로 한다.

- 최고관리자  
- 운영관리자  
- CS관리자  
- 정산관리자  
- 마케팅관리자  

## 4. 공통 로그 구조

아래 유형 등으로 **구조화된 로그**를 일관되게 저장한다.

- 로그인  
- 거래  
- 수정  
- 상태변경  
- 정산  
- 관리자행동  

## 5. 금액 계산은 서버 기준

모든 금액·수량·수수료 계산은 **서버가 단일 진실 소스**이며, 프론트 계산값은 신뢰하지 않는다.

## 6. API 공통 규칙

- 응답 형식 통일  
- 에러코드 통일  
- 인증 방식 통일  

## 7. 모바일 우선 UI

UI는 **모바일 우선** 기준으로 개발한다.

## 8. DB 확장 가능 설계

추후 다음 영역을 추가할 수 있도록 스키마·도메인 경계를 넓게 잡는다.

- 거래소  
- P2P  
- 모의투자  
- 카드  
- DAO  
- 토큰  

## 9. 기존 기능 삭제 금지

기능 추가 시 **기존 기능을 테스트한 뒤** 진행한다.

## 10. 테스트 리포트

개발 후 **반드시 테스트 리포트**를 작성한다.
